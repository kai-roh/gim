# GIM — Geometry based Integrated Modeling

> 초고층 빌딩 기획 컨텍스트를 입력하면, 사용자가 선택한 거장 건축가 AI 패널이 토론을 통해 설계안을 확정하고, 수직 노드 그래프가 Rhino 3D 모델로 실시간 생성되며, 평가 모델의 피드백이 다시 토론장으로 돌아오는 순환 설계 시스템

---

## 개요

GIM은 초고층 빌딩 **기획설계 단계**를 위한 AI 기반 의사결정 시스템이다. BIM이 실시설계/시공 단계의 정밀 문서화 도구라면, GIM은 아이디어 단계에서 다양한 건축적 관점을 충돌시키고 수렴시키는 **설계 프로토콜**이다.

| | BIM | GIM |
|---|---|---|
| 단계 | 실시설계/시공 | 아이디어/기획설계 |
| 데이터 | 자재, 구조, MEP | 층별 프로그램, 공간 관계, 수직 동선 |
| 정밀도 | mm 단위 | 매스 레벨 (층 단위 그리드) |
| 목적 | 시공 문서화 | 설계 의사결정 지원 |
| AI 활용 | 간섭 체크, 물량 산출 | 배치 추론, 건축가 토론, 피드백 루프 |

## 핵심 기능

### 1. 건축가 토론장 (Architect Forum)

20인의 거장 건축가 AI 클론 풀에서 2~5명을 선택하여 토론 패널을 구성한다. 서로 다른 건축 철학을 가진 건축가들이 하나의 초고층 프로젝트를 두고 4단계 토론 프로토콜로 합의안을 도출한다.

```
[Phase 1: 발제]  →  [Phase 2: 교차 비평]  →  [Phase 3: 수렴]  →  [Phase 4: 확정]
                        ↑                                              │
                        └──────────── 평가 피드백 수신 시 재개 ──────────┘
```

### 2. 수직 노드 그래프

건축가 토론 합의 결과가 지하~옥상까지의 수직 노드 그래프로 자동 변환된다. Autodesk Building-GAN(ICCV 2021)의 3-그래프 구조를 초고층에 맞게 적용했다.

```
[옥상] ──── 헬리패드 ─── 전망대 ─── 기계실
  │
[고층 50~60F] ── 스카이라운지 ─── 호텔객실 ─── 코어
  │
[중층 25~38F] ── 오피스 ─── 코어 ─── 회의실
  │
[스카이로비 24F] ── 스카이로비 ─── 셔틀EV
  │
[저층 1~5F] ── 로비 ─── 리테일 ─── 공공보이드
  │
[지하 B1~B2] ── 주차 ─── 기계실
```

노드 function enum 30+개 (CORE, OFFICE, HOTEL, RETAIL, PUBLIC, SOCIAL, MECHANICAL, PARKING, STRUCTURAL)와 8종의 엣지 타입으로 초고층 설계 관계를 정형화한다.

### 3. Rhino MCP 3D 모델링

수직 노드 그래프가 RhinoMCP를 통해 3D 매스로 실시간 변환된다. W2(노드 그래프)와 W3(3D 뷰포트)는 양방향으로 동기화되며, 노드 편집 이벤트는 delta 방식으로 전송되어 변경된 부분만 재생성된다.

### 4. 평가 모델 (Evaluation Engine)

3D 모델이 생성될 때마다 자동으로 7개 차원을 평가하고 건축가 토론장에 피드백을 전달한다.

| 차원 | 평가 항목 | 방식 |
|---|---|---|
| 구조 안정성 | 코어 비율, 종횡비, 풍하중 개략 검토 | 룰 기반 + LLM |
| 환경 성능 | 조망률, 일조, 바람, 에너지 등급 | 파라미터 기반 |
| 법규 적합도 | 용적률, 건폐율, 피난 규정 | 룰 기반 |
| 경제성 | 전용률, 프리미엄 층 비율 | 산식 기반 |
| 도시 맥락 | 공공 기여도, 스카이라인 조화 | LLM 추론 |
| 3D 형태 | 매스 비례, 입면 일관성 | 3D 분석 룰 + MCP |
| 그래프 연결성 | 프로그램 인접 관계 유지율 (Connectivity Accuracy) | 그래프 분석 |

### 5. 순환 피드백 루프

```
[건축가 토론] → [수직 노드 그래프] → [Rhino 3D] → [평가 모델]
      ↑                                                    │
      └──────────────── 피드백 리포트 ─────────────────────┘
```

루프는 평가 점수가 임계값 이상이거나 사용자가 "확정"할 때 종료된다.

## 기술 스택

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Next.js |
| 그래프 시각화 | D3.js (수직 레이아웃 커스텀) |
| 3D 웹 뷰어 | Three.js |
| 실시간 동기화 | WebSocket |
| 노드 그래프 저장 | NetworkX |
| LLM | Claude API (function calling) |
| 토론 엔진 | Claude multi-turn + system prompt per clone |
| 평가 모델 | Python 룰 엔진 + Claude 추론 |
| 3D 모델링 연결 | RhinoMCP |
| 데이터 포맷 | JSON, YAML, CSV, Markdown |

```
├── packages/
│   ├── core/
│   │   ├── forum/               # 토론 엔진 (멀티에이전트 오케스트레이션)
│   │   ├── graph/               # 노드 그래프 스키마 + 변환 로직

│       └── sync.ts              # 양방향 동기화
│
├── data/
│   ├── architects/              # 건축가 클론 프로필 (YAML, 20개)
│   └── rules/                   # 초고층 설계 룰셋 (YAML)
│
└── rhino/                       # Rhino/Grasshopper 스크립트
    ├── mass_generator.py
    ├── delta_updater.py
    └── form_analysis.py
```
