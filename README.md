# GIM — Geometry based Integrated Modeling

> 기업 사옥 프로젝트의 기획 컨텍스트를 입력하면, 사용자가 선택한 건축가 AI 패널과 전문가(법률/규제, 구조) 패널이 다자 토론과 전문 검증을 통해 설계안을 확정하고, 수직 노드 그래프가 웹 기반 3D 매스로 실시간 생성되며, 평가 모델의 피드백이 다시 토론장으로 돌아오는 순환 설계 시스템

---

## 개요

GIM은 기업 사옥 **기획설계 단계**를 위한 AI 기반 의사결정 시스템이다. BIM이 실시설계/시공 단계의 정밀 문서화 도구라면, GIM은 아이디어 단계에서 다양한 건축적 관점을 충돌시키고 수렴시키는 **설계 프로토콜**이다.

| | BIM | GIM |
|---|---|---|
| 단계 | 실시설계/시공 | 아이디어/기획설계 |
| 데이터 | 자재, 구조, MEP | 층별 프로그램, 공간 관계, 브랜드 정체성 |
| 정밀도 | mm 단위 | 매스 레벨 (층 단위 그리드) |
| 목적 | 시공 문서화 | 설계 의사결정 지원 |
| AI 활용 | 간섭 체크, 물량 산출 | 배치 추론, 건축가+전문가 토론, 피드백 루프 |

### 왜 기업 사옥인가?

- **브랜드 정체성 표현**: 젠틀몬스터 사옥, Apple Park, Google Bjarke처럼 기업의 철학이 건축으로 구현
- **층별 다양성**: 1층 리테일은 Hadid 스타일, 사무공간은 Mies 스타일, 옥상은 Ando 스타일처럼 각 층마다 서로 다른 건축적 언어가 공존
- **설계 자유도**: 중저층(5~20층) 규모에서 형태적 실험이 가능
- **실제 수요**: 스타트업/테크 기업의 사옥 프로젝트가 증가하는 트렌드

## 핵심 기능

### 1. 건축가 토론장 (Architect Forum)

20인의 거장 건축가 AI 클론 풀에서 2~5명을 선택하여 토론 패널을 구성한다. 서로 다른 건축 철학을 가진 건축가들이 하나의 프로젝트를 두고 4단계 토론 프로토콜로 합의안을 도출한다.

```
[Phase 1: 발제]  →  [Phase 2: 교차 비평]  →  [Phase 3: 수렴]  →  [Phase 4: 확정]
                         ↑                                              │
                         └──────────── 평가/전문가 피드백 수신 시 재개 ────┘
```

### 2. 전문가 검증 패널 (Expert Review)

건축가 토론과 병행하여 전문가 패널이 자동으로 참여해 설계안의 실현 가능성을 검증한다.

| 전문가 | 역할 |
|---|---|
| 법률/규제 전문가 | 용적률, 건폐율, 법규 적합성 검토 → 수정 권고 |
| 구조 전문가 | 구조 실현 가능성, 하중, 시스템 적합성 검토 |

전문가 패널의 "승인" 판정이 루프 종료 조건 중 하나다.

### 3. 수직 노드 그래프

건축가 토론 합의 결과가 지하~옥상까지의 수직 노드 그래프로 자동 변환된다. Autodesk Building-GAN(ICCV 2021)의 3-그래프 구조를 적용했다.

```
[옥상] ──── 옥상정원 ─── 기계실
  │
[상층부] ── 임원층 ─── 회의공간 ─── 코어
  │
[중층부] ── 업무공간 ─── 코어 ─── 협업공간
  │
[저층부] ── 로비 ─── 리테일 ─── 공공보이드
  │
[지하] ──── 주차 ─── 기계실
```

노드 function enum과 8종의 엣지 타입(수직 연속성, 프로그램 인접, 기능 종속 등)으로 설계 관계를 정형화한다.

### 4. 웹 기반 3D 매스 (Three.js)

수직 노드 그래프가 Three.js 기반 3D 매스로 브라우저에서 실시간 변환된다. W2(노드 그래프)와 W3(3D 뷰포트)는 양방향으로 동기화된다.

### 5. 평가 모델 (Evaluation Engine)

3D 모델이 생성될 때마다 자동으로 평가하고 건축가 토론장과 전문가 패널에 피드백을 전달한다.

| 차원 | 평가 항목 | 방식 |
|---|---|---|
| 구조 실현성 | 구조 시스템 적합성, 하중 검토 | 룰 기반 + LLM |
| 법규 적합도 | 용적률, 건폐율, 피난 규정 | 룰 기반 |
| 브랜드 정체성 | 기업 철학과 공간 언어의 정합성 | LLM 추론 |
| 공간 품질 | 채광, 조망, 공간 비례, 동선 효율 | 파라미터 기반 |
| 그래프 연결성 | 프로그램 인접 관계 유지율 (Connectivity Accuracy) | 그래프 분석 |

### 6. 순환 피드백 루프

```
[건축가 토론] → [전문가 검증] → [수직 노드 그래프] → [Three.js 3D] → [평가 모델]
      ↑                                                                      │
      └──────────────────────── 피드백 리포트 ────────────────────────────────┘
```

루프 종료 조건: 평가 점수 임계값 달성 / 전문가 패널 승인 / 사용자 확정 명령

## 기술 스택

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Next.js |
| 그래프 시각화 | D3.js (수직 레이아웃 커스텀) |
| 3D 뷰어 | Three.js |
| 실시간 동기화 | WebSocket |
| LLM | Claude API (function calling) |
| 토론 엔진 | Claude multi-turn + system prompt per clone |
| 평가 모델 | 룰 기반 + Claude 추론 |
| 데이터 포맷 | JSON, YAML, Markdown |
